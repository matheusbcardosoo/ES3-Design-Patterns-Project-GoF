Padrões de Projeto Implementados

Este sistema de Clínica Veterinária implementa diversos padrões de projeto:

  • MVC (Model-View-Controller): Separação clara de responsabilidades entre modelo (domain, repositories), visualização (templates Thymeleaf) e controle (controllers).
  • DAO (Data Access Object): Utilizado para abstrair e encapsular o acesso aos dados através das classes AnimalDAO, ClienteDAO, ConsultaDAO e VeterinarioDAO.
  • Singleton: Utilizado na classe ClinicaVeterinaria para garantir instância única.
  • Factory: Implementado para criação de diferentes tipos de usuários e animais.
  • Builder: Utilizado para construir objetos complexos como Cliente, Animal e Consulta.
  • Adapter: Implementado para integração com sistema de pagamento externo.
  • Facade: Simplifica o acesso a subsistemas complexos através do AgendamentoFacade.
  • Strategy: Utilizado para implementar diferentes estratégias de validação de objetos de domínio, permitindo validações específicas para cada tipo de objeto sem depender de validações HTML.
  • Command: Implementado para encapsular operações relacionadas às consultas, permitindo registrar, executar e desfazer operações como agendamento, cancelamento e busca de consultas.
Informações Técnicas

Esta aplicação foi construída utilizando:

  • Spring Boot: Framework Java para desenvolvimento de aplicações
  • Thymeleaf: Engine de templates para front-end
  • MongoDB: Banco de dados NoSQL para persistência de dados
  • Docker: Contêinerização do banco de dados
  • Bootstrap: Framework CSS para estilização

A arquitetura segue princípios de Clean Code e SOLID, com separação clara de responsabilidades.
